## 未授权漏洞

**未授权访问漏洞，简单来说，就是系统因为身份验证或访问控制措施失效，允许了本不该有权限的人访问了敏感资源或执行了关键操作**

- **未授权访问**
- **水平越权**
- **垂直越权**





## Redis未授权访问

redis是一个开源的使用ANSI C语言编写、支持网络、可基于内存亦可持久化的日志型、Key-Value数据库，并提供多种语言的API。和Memcached类似。redis支持存储的value类型相对更多，包括string(字符串)、list(链表)、set(集合)和zset(有序集合)。

### 漏洞简介

Redis安装后默认情况下，会绑定在 0.0.0.0:6379 ，如果没有进行采用相关的策略，比如添加防火墙规则避免其他非信任来源 ip 访问等，这样将会将 Redis 服务暴露到公网上，如果在没有设置密码认证（一般为空）的情况下，会导致任意用户在可以访问目标服务器的情况下 未授权访问Redis以及读取 Redis 的数据 。



### 漏洞危害

攻击者在未授权访问 Redis 的情况下，利用 Redis 自身的提供的config 命令，可以进行写文件操作，攻击者可以成功将自己的ssh公钥写入目标服务器的 /root/.ssh 文件夹的authotrized_keys 文件中，进而可以使用对应私钥直接使用ssh服务登录目标服务器。
