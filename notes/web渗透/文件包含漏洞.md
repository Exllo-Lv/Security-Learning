# 文件包含漏洞

什么叫包含呢，以PHP为例，我们常常把可重复使用的函数写入单个文件中，在使用该函数时，直接调用此文件，而无需再次编写函数，这个过程就叫做包含。

## 包含函数

```php
require()#找不到被包含的文件时会产生致命错误，并停止脚本运行。
require('database.php')
include()#找不到被包含的文件时只会产生警告，脚本将继续运行。
include('database.php')
include_once()与include()类似，唯一区别是如果该文件中的代码已经被包含，则不会再次包含。
require_once()与require()类似，唯一区别是如果该文件中的代码已经被包含，则不会再次包含。
```

其他语言:

PHP：include() 、include_once()、require()、require_once()
JSP/Servlet：ava.io.file()、java.io.filereader()
ASP：include file、include virtual



## 漏洞成因

程序在通过函数包含文件时，由于传入的包含文件是用户可控的且没有经过严格的校验，从而可以去包含一个恶意文件，就可能导致意外的文件泄露甚至服务器沦陷。

示例:

```php
<?php
$file = $_GET['page']; // 用户完全控制 'page' 参数
include($file);        // 直接包含了用户指定的文件
?>
```

攻击者可以构造 `?page=../../etc/passwd` 来读取服务器上的敏感文件，这就是最典型的利用方式。



## 漏洞分类



| 类型             | 英文简称 | 定义                               | 核心触发条件                                                 |
| :--------------- | :------- | :--------------------------------- | :----------------------------------------------------------- |
| **本地文件包含** | **LFI**  | 包含并执行**服务器本地**的文件。   | 1. 存在文件包含函数，且参数用户可控。 2. 能绕过路径限制（如使用 `../` 进行目录遍历）。 |
| **远程文件包含** | **RFI**  | 包含并执行**远程服务器**上的文件。 | 1. 存在文件包含函数，且参数用户可控。 2. PHP配置中 `allow_url_include` 和 `allow_url_fopen` 均需设置为 `On`。 |





### 常见利用方式

1. **利用PHP伪协议**：PHP内置的伪协议为漏洞利用提供了丰富的手段。
   - **`php://filter`**：用于**读取文件源码**。它可以将文件内容进行Base64编码后返回，从而避免PHP代码被直接执行，让攻击者能获取到敏感文件的完整源码。
     - **示例**：`?page=php://filter/read=convert.base64-encode/resource=config.php`
   - **`php://input`**：用于**执行任意代码**。在 `allow_url_include` 开启时，攻击者可以在POST请求体中发送PHP代码，代码会被直接执行。
   - **`data://`**：类似 `php://input`，可直接在URL中嵌入经过编码的PHP代码执行。
2. **包含日志文件 (Log Poisoning)**：如果无法直接上传文件，攻击者会尝试包含Web服务器（如Apache、Nginx）的访问日志或错误日志。攻击者先发送一个包含恶意PHP代码的请求（如 `User-Agent: <?php phpinfo();?>`），该请求会被记录在日志中。随后，通过LFI漏洞包含这个日志文件，恶意代码就会被执行，从而获得Webshell。
3. **包含Session文件**：与日志文件类似，如果攻击者能控制Session文件的部分内容（如用户名），也可以利用LFI去包含Session文件来执行代码。





## 防御措施

防御文件包含漏洞的核心原则是**永远不要信任用户的输入**。以下是几种有效的防御策略：

- **使用白名单策略**（最推荐）：明确规定只允许包含哪些文件，对于白名单之外的任何输入，直接拒绝。

  php

  ```php
  $allowed_pages = ['home', 'about', 'contact'];
  if (in_array($_GET['page'], $allowed_pages)) {
      include($_GET['page'] . '.php');
  } else {
      // 报错或跳转到默认页面
  }
  ```

- **禁用危险功能**：在 `php.ini` 中**务必关闭** `allow_url_include`，如果业务不需要，也建议关闭 `allow_url_fopen`。这是防御远程文件包含最根本的方法。

- **路径限制**：通过设置 `open_basedir` 配置项，将PHP可操作的文件目录限制在项目根目录内，可以有效防御目录遍历攻击，即使包含漏洞存在也无法访问系统敏感文件。

- **输入过滤**：对用户输入的 `../`、`..\`、`http://`、`https://` 等危险字符进行过滤。但要注意，简单的字符串替换（如 `str_replace`）容易被双写等方式绕过，应结合白名单使用。
