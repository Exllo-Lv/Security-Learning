# 信息收集
## 什么是信息收集
渗透测试的本质是信息收集，在渗透测试中信息收集的质量直接关系到渗透测试成攻与否。在对系统进行渗透测试前的信息收集是通过各种方式获取所需要的信息，收集的信息越多对目标进行渗透的优势越有利。

通过利用获取到的信息对系统进行渗透。只有掌握了足够多的目标信息才能更好的对目标系统进行渗透。正所谓知彼知己百战不殆。

## 为什么要做信息收集 
信息收集对于渗透测试前期来说非常重要。只有我们掌握了目标网站或主机足够多的信息之后，才更好的进行渗透测试。同时测试漏洞的资产面也会增加。那么意味着挖到漏洞的几率就大些。

## 怎么做信息收集
信息收集就是尽可能的搜集目标的信息，主要是收集服务器的配置信息和网站的敏感信息包括：（域名信息、子域名信息、目标架构、IP地址、端口、脚本语言、中间件、Whois信息、邮箱信息、源码信息）等等看似并不起眼的一些信息都需要进行搜集，这些看似微乎其微的信息，对于渗透测试而言就关乎到成功与否。

信息收集的重要性：信息收集是渗透测试的最重要的阶段，根据收集的有用信息，可以提高我们渗透测试的成功率。

为什么信息收集：扩大测试资产面，获取更多目标的信息。提高渗透测试成功率。

## 信息收集分类 
从信息收集方式分类可将信息收集分为两大类： 主动信息收集、被动信息收集。

### 主动信息收集
通过主动扫描目标主机或网站，对目标进行探测性扫描获取目标信息。这种方式能够获取到更多、更准确的信息。但是缺点是通过主动信息收集容易留下明显的攻击访问痕迹，易发现和被溯源

```shell
端口扫描
192.168.174.1 1-65535
主动向对方机器发送探测数据包，探测对方开放了什么端口
```

### 被动信息收集 
通过搜索引擎等公开渠道进行相关信息的搜索收集，这种方式不直接与目标建立访问连接，避免了留下访问和收集痕迹。被动收集的优缺点正好与主动收集的优缺点相反，被动信息收集不会留下访问痕迹，避免被溯源发现，但是收集到的信息依赖于搜索引擎，准确性和质量无法保证。

```shell
端口扫描
						通过三方网站，去搜索该ip地址开放了哪些端口
```

信息收集也分为 资产收集、信息收集

+ 资产收集
        * 针对企业资产进行收集、梳理。
+ 信息收集
        * 对服务器和Web站点等进行信息收集。

# 资产收集 
## 什么是资产？
在渗透测试中，"资产" 指的是企业或目标 环境中的各种信息技术资源，这些资源可能是攻击者潜在的目标。

<font style="background-color:#cef1f7;">1.域名、2.子域名、3.IP地址 、4.Web应用、5.APP、6.小程序 、7.服务器、8.旁站/C段、9.账户/邮箱信息等。</font>

## 为什么需要资产收集
渗透测试的目标通常是发现和利用这些资产中的潜在漏洞和安全弱点 。

<font style="background-color:#cef1f7;">在实际的测试过程中，客户通常会给我们部分主域名或（公司名）。此时 就需要对企业的资产进行梳理，随着企业内部的业务不断壮大，各种业务 平台和管理系统越来越多，很多单位往往存在着 隐形资产 这些 “隐形资产”通常被管理员所遗忘，长时间无人维护，可能导致存在较多的已知漏洞。 </font>

		扩大攻击面，发现一些隐形资产。 

## 小结
渗透测试中，测试人员需要尽可能多的去收集目标的信息。资产收集和信息收集的深度，决定了你发现安全漏洞的几率有多大。

所谓资产收集就是从公司或主域名出发，尽可能去收集关于该公司/主域名的一切信息。

为啥收集子域名

假设目标资产数量庞大，一般来说主站的安全性相对较高，防护也相对很严格。

而不常用的子站或上线不久的站点，可能安全方面考虑还没有很周全，可能成为目标系统的脆弱点。因此从子域名开始测试便成了很好的选择，子域名往往在防护上会相对更加松懈。同一组织采用相同应用搭建多个服务的可能性很大，以及补丁的情况也可能大致相同，因此，存在相同漏洞的概率非常大。

CDN（内容分发网络），实现方式就是通过在互联网各处放置 节点服务器 所构成的在现有的互联网基础之上的一层 智能虚拟网络 ，CDN核心的就是使用户可就近访问网络,取得所需内容，解决网络拥挤的状况，提高用户访问网站的响应速度或者用户下载速度。一般来说,网站开启 CDN 之后，会根据用户所在地的不同访问 CDN<font style="color:rgb(0,0,0);">的节点服务器，并不直接访问源服务器，这样可以减少网站服务器宽带资源,降低服务器压力，可以提升用户体验。</font>

PS：这便是大家在不同的地区Ping相同的域名反馈IP不一样的原因。其次，由于CDN 节点的阻挡防护，可以更好的保护原服务器的安全。

# <font style="color:rgb(0,0,0);">公网IP的C段有用吗</font>
<font style="color:rgb(31, 31, 31);">公网 IP 的 C 段</font>**<font style="color:rgb(31, 31, 31);">非常有用</font>**<font style="color:rgb(31, 31, 31);">，甚至可以说它是互联网基础架构、网络安全以及资产管理中一个极其关键的划分单位。</font>

<font style="color:rgb(31, 31, 31);">在日常交流中，我们常说的“C 段”通常是指一个 </font>`**<font style="color:rgb(68, 71, 70);">/24</font>**`**<font style="color:rgb(31, 31, 31);"> 的子网</font>**<font style="color:rgb(31, 31, 31);">（即 256 个 IP 地址，从 </font>`<font style="color:rgb(68, 71, 70);">x.x.x.0</font>`<font style="color:rgb(31, 31, 31);"> 到 </font>`<font style="color:rgb(68, 71, 70);">x.x.x.255</font>`<font style="color:rgb(31, 31, 31);">），而不仅仅是早期传统分类网络中的 C 类地址。</font>

<font style="color:rgb(31, 31, 31);">它的主要用途可以体现在以下几个核心领域：</font>

### <font style="color:rgb(31, 31, 31);">渗透测试与信息收集（核心应用场景）</font>
<font style="color:rgb(31, 31, 31);">在网络安全和漏洞挖掘领域，</font>**<font style="color:rgb(31, 31, 31);">“C段探测”</font>**<font style="color:rgb(31, 31, 31);"> 是极其标准且重要的一步。</font>

+ **<font style="color:rgb(31, 31, 31);">扩大攻击面（旁站/同段打点）：</font>**<font style="color:rgb(31, 31, 31);"> 企业通常会向运营商连续租用一段 IP，或者直接购买一个 C 段。如果目标系统（比如一个防御严密的 Web 主站）位于 </font>`<font style="color:rgb(68, 71, 70);">198.51.100.10</font>`<font style="color:rgb(31, 31, 31);">，那么这个 C 段（</font>`<font style="color:rgb(68, 71, 70);">198.51.100.1</font>`<font style="color:rgb(31, 31, 31);"> - </font>`<font style="color:rgb(68, 71, 70);">254</font>`<font style="color:rgb(31, 31, 31);">）上的其他 IP 极有可能也属于该企业。</font>
+ **<font style="color:rgb(31, 31, 31);">发现薄弱资产：</font>**<font style="color:rgb(31, 31, 31);"> 这些同段的 IP 上，经常会暴露出企业遗忘的测试环境、没有开启 WAF 的老旧业务系统、未授权访问的数据库，或者是配置不当的 VPN 网关。通过拿下这些安全防护较弱的“邻居”，就可以作为跳板，进一步向目标核心内网渗透。</font>

### <font style="color:rgb(31, 31, 31);">路由宣告（BGP）的最小单位</font>
<font style="color:rgb(31, 31, 31);">从底层网络通信的角度来看，公网 C 段是互联网路由表（全局 BGP）能够接受的</font>**<font style="color:rgb(31, 31, 31);">最小宣告单位</font>**<font style="color:rgb(31, 31, 31);">。</font>

:::color2
### <font style="color:rgb(31, 31, 31);">一、 什么是“互联网路由表”？</font>
<font style="color:rgb(31, 31, 31);">你可以把互联网路由表（Internet Routing Table，通常指 BGP 全局路由表）理解为</font>**互联网的“全球导航地图”或“国家邮政编码簿”**<font style="color:rgb(31, 31, 31);">。</font>

<font style="color:rgb(31, 31, 31);">当你在电脑上输入一个网址或访问一个 IP 时，你的数据包（快递包裹）会被发送给你的运营商（比如电信、联通）。运营商的核心路由器拿到了这个包裹，它怎么知道该往哪里发呢？</font>

<font style="color:rgb(31, 31, 31);">它就是通过查阅</font>**互联网路由表**<font style="color:rgb(31, 31, 31);">来决定的。</font>

+ <font style="color:rgb(31, 31, 31);">这张表里记录着全球所有公网 IP 地址段的“归属地”和“转发路径”。</font>
+ <font style="color:rgb(31, 31, 31);">比如表里会写着：“想要把数据送到 </font>`<font style="color:rgb(31, 31, 31);">198.51.100.x</font>`<font style="color:rgb(31, 31, 31);"> 这个网段，你需要先把数据交给中国电信，中国电信再交给美国 AT&T，最后到达目标”。</font>
+ <font style="color:rgb(31, 31, 31);">全球各大运营商和大型互联网公司（如谷歌、腾讯）的核心路由器之间，无时无刻不在互相交换这本“地图”的最新信息，确保大家手里的地图都是最新的。</font>

### 二、 什么是“宣告”（Announcement）？
在互联网术语中，“宣告”（BGP Advertisement）就是**向全网广播自己的位置和拥有的 IP 资产**。

想象一下，你开了一家新的快递分拣中心（你的网络），并且你向有关部门申请了一批邮政编码（公网 IP 段）。为了让全国的快递员都能把属于你的包裹送过来，你必须向邮政总局**大喇叭广播**：“大家好！邮编从 100000 到 100099 的包裹，请全部发给我！”

在互联网中，企业或运营商通过 BGP（边界网关协议）向全球其他核心路由器发送广播：“这批 IP 地址是我负责的，请把发往这些 IP 的数据包都交给我”。**这个广播的动作，就叫“宣告”。**

### 三、 为什么 C 段（/24）是最小“宣告单位”？
“宣告单位”就是你每次向全网广播时，**允许播报的最小 IP 块的大小**。

为什么国际互联网规定，最小只能宣告一个 C 段（也就是一个 `/24` 的子网，包含 256 个连续的 IP 地址）呢？

+ **为了防止“路由表爆炸”（核心路由器的内存不够用）**
    - 想象一下，如果地图导航精确到了“全世界每一个人的具体卧室床位”，这个地图的数据量会有多恐怖？导航仪瞬间就会死机。
    - 如果允许大家宣告单个 IP（`/32`），或者十几个 IP，全球互联网路由表里的条目可能会膨胀到几十亿条。全世界最顶级的核心路由器也扛不住这么庞大的数据量，查询速度会巨幅下降，整个互联网就会瘫痪。
+ **国际通用规则的妥协**
    - 为了在“节省路由器内存”和“灵活分配 IP”之间寻找平衡，全球的互联网骨干网运营商达成了一个默契的硬性规定：**任何人在公网上宣告的 IP 块，最小不能低于 256 个 IP（即一个 C 段）**。
    - 如果你在 BGP 路由器上配置宣告一个只有 16 个 IP 的网段，全球其他的骨干路由器在收到这个广播时，会直接把它**丢弃（过滤）掉**，根本不把它写进全球导航地图里。结果就是，外面的人永远找不到你这 16 个 IP。

总结来说，互联网路由表就是全球路由器的**“寻路指南”，**而宣告单位就是这本指南里允许登记的**“最小地块面积”**。

:::

+ **<font style="color:rgb(31, 31, 31);">地址可携带性：</font>**<font style="color:rgb(31, 31, 31);"> 如果一个企业想要拥有“属于自己”的公网 IP，并且在更换运营商（比如从电信换到联通）时 IP 不变，他们就必须自己去 RIR（如 APNIC）申请 IP。而申请和在公网上宣告的最小门槛就是一个 C 段（</font>`/24）。</font>
+ **<font style="color:rgb(31, 31, 31);">防止路由表爆炸：</font>**<font style="color:rgb(31, 31, 31);"> 如果允许单独宣告一个 </font>`/32`<font style="color:rgb(31, 31, 31);">（单个 IP）或 </font>`>/28`<font style="color:rgb(31, 31, 31);">，全球核心路由器的路由表会瞬间崩溃。因此，拥有一个独立的公网 C 段，是构建企业级独立自治系统（AS）的基础。</font>

### <font style="color:rgb(31, 31, 31);">威胁防御与黑名单溯源</font>
<font style="color:rgb(31, 31, 31);">在安全防御（如应急响应、防火墙策略）层面，C 段常常被作为一个整体进行评估和处置。</font>

+ **<font style="color:rgb(31, 31, 31);">区域封禁：</font>**<font style="color:rgb(31, 31, 31);"> 如果服务器遭到大规模的 SSH 暴力破解或 DDoS 攻击，防御方经常会发现攻击源虽然 IP 不断变化，但都集中在某几个特定的 C 段（通常是某些云厂商或特定地区的“肉鸡”网段）。此时，直接在防火墙上封禁整个 </font>`/24<font style="color:rgb(31, 31, 31);"> C 段是最简单高效的防御策略。</font>
+ **<font style="color:rgb(31, 31, 31);">信誉评估：</font>**<font style="color:rgb(31, 31, 31);"> 如果一个 C 段内有大量 IP 被用于发垃圾邮件或做恶意扫描，这个 C 段的整体“IP 信誉”就会极低，从而被各大安全厂商列入黑名单。</font>

### <font style="color:rgb(31, 31, 31);">网站 SEO 与“站群”业务</font>
<font style="color:rgb(31, 31, 31);">在搜索引擎优化（SEO）领域，IP 的 C 段分布对网站权重有直接影响。</font>

+ <font style="color:rgb(31, 31, 31);">搜索引擎如果发现很多相互链接的网站都在同一个 C 段下，会判定这些网站可能是为了刷排名而建立的“站群”，从而进行降权处罚。因此，做此类业务的人通常会特意购买“多 C 段”的公网 IP 服务器来规避检测。</font>
