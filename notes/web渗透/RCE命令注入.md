# RCE(命令执行与代码执行)

Remote Command Execution（命令执行）

Remote Code Execution（代码执行）

 在 Web 应用中有时候程序员为了考虑灵活性、简洁性,会在代码调用代码或命令执 行函数去处理。比如当应用在调用一些能将字符串转化成代码的函数时,没有考 虑用户是否能控制这个字符串,将造成代码执行漏洞。同样调用系统命令处理,将造 成命令执行漏洞。  

| 特征维度         | 远程代码执行 (Code Execution)                   | 远程命令执行 (Command Execution)                             |
| :--------------- | :---------------------------------------------- | :----------------------------------------------------------- |
| **执行目标**     | 执行应用层的**脚本代码**（如PHP、Python、Java） | 执行操作系统底层的**系统命令**（如Linux Shell、Windows cmd） |
| **常见函数示例** | `eval()`、`assert()`、`preg_replace()`          | `system()`、`exec()`、`shell_exec()`、`passthru()`           |
| **攻击举例**     | 注入 `phpinfo();` 让服务器执行并返回信息        | 注入 `; whoami` 让服务器执行并返回当前用户名                 |
| **危害关系**     | 通常可以借此**调用系统命令**,进而升级为命令执行 | 是获取服务器控制权更直接的方式,两者常被统称为RCE             |



## 命令执行

该漏洞的出现是由于应用系统从设计上需要给用户提供指定的远程命令操作的接口,例如在防火墙的WEB界面中会存在一个故障排除功能,里面就会存在类似Ping操作的界面,若设计者未针对这类功能进行严格的控制检测,则可能导致攻击者提交恶意命令,从而控制后台,控制服务器。



### 漏洞原理

PHP应用系统在调用命令执行函数执行命令的时候,如果用户的输入数据作为系统命令函数的参数拼接到命令中并执行,且在没有过滤用户的输入的情况下,就会造成命令执行漏洞.



### 危险函数

| 函数名称           | 典型用法                 | 为何危险                                                    |
| :----------------- | :----------------------- | :---------------------------------------------------------- |
| **`system()`**     | `system("ping $ip")`     | 直接执行命令,并**实时输出**结果给浏览器,攻击者最爱          |
| **`exec()`**       | `exec("ping $ip", $out)` | 执行命令,但不输出结果,而是**存到数组**里,隐蔽性更高         |
| **`shell_exec()`** | `shell_exec("ping $ip")` | 使用反引号 ``` 是它的别名,执行命令并**返回完整结果**字符串. |
| **`passthru()`**   | `passthru("ping $ip")`   | 执行命令并直接**输出原始二进制或文本**,常用于和外部程序交互 |
| **`popen()`**      | `popen("ping $ip", 'r')` | **建立双向管道**,命令执行是异步的,但同样危险.               |

方式三：反弹**Shell**

简述：反弹shell，就是攻击机监听在某个TCP/UDP端口为服务端，目标机主动发起请求到攻击机监听的端口，并将其命令行（shell）转到攻击机。

攻击者：nc -lvnp 8000 监听攻击者本机的8000 端口，等待连接 -> 接收/bin/bash 

受害者：/bin/bash sh ---------> 攻击者ip：8000

执行反弹 shell 的命令有许多。 

反弹 shell 因为是从受害者，反向连接远程服务器，请求是从内部到外部，所以防火墙防火墙是不会进行拦截。



#### **NC**（**netcat**） 

nc是netcat的简写也称瑞士军刀。因为它短小精悍、功能实用，被设计为一个简单、可靠的网络工具。

**NC**的作用 

- 实现任意TCP/UDP端口的监听，nc可以作为Server以TCP或UDP形式监听指定端口 nc -lvnp 4444 

- 端口扫描，NC可以作为client发起TCP或UDP连接 

- 远程控制等操作

**NC**常用参数

-l 监听模式，用于入站连接 

-p 指定监听本地的端口 

-n 后面跟上IP地址，直接解析IP不解析域名，速度更快 

-v 输出连接的详细信息，输入vv可以看到更详细的信息 

-z 端口扫描，默认为tcp模式 

-u 端口扫描，udp方式 

-c shell命令，通常为-c bash返回一个bash shell

常用操作

nc -lvnp 4444 

nc -lvnp 4444 监听本机4444端口（作为服务端） ，等待客户段发送数据到服务端4444端口 

nc -nv 192.168.8.110 8888 主动连接对方8888端口获得shell 

nc -nvz 192.168.174.1 1-1000 端口扫描
